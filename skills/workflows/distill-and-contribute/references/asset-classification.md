# Asset Classification

## 先判断“这是什么”

### 放到 `skills/roles`

当内容主要是：

- 岗位使命与边界
- 默认视角
- 优先级规则
- 关键对象沟通差异
- 常见误区
- 调用 workflow / method / pack 的规则

### 放到 `skills/workflows`

当内容主要是：

- 最低输入
- 动作顺序
- 输出结构
- 检查点
- 失败场景

### 放到 `knowledge-packs`

当内容主要是：

- 公开事实
- 公开产品知识
- 公开来源
- 可被客户或外部直接引用的说明

### 放到 `methods`

当内容主要是：

- 可跨岗位复用的判断法
- 推进法
- 复盘法
- 汇报法
- 管理法

## 一条硬规则

如果内容主要在回答“这个岗位通常先看什么、先判断什么、先避开什么”，它更可能属于 `skills/roles`。
如果内容主要在回答“这个场景要收什么输入、按什么顺序推进、输出什么”，它更可能属于 `skills/workflows`。
如果一段内容离开特定岗位和机构后仍然可复用，它更可能属于 `methods`，而不是 `skills` 或 `knowledge-packs`。
