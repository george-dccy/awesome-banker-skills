# 示例

## 示例一：季度经营分析汇报（同步型+背景先行）

**场景**：季度结束，向分管领导汇报经营情况，需要让领导知道结果，不需要当场拍板。

**判断过程**：

1. 判断类型：不汇报什么都不会卡住，但领导需要了解情况 → 同步型
2. 决策点数量：1个（了解情况，不需要做决定）→ 同步型结构
3. 质疑预判：无强质疑点 → 结论先行
4. 资源需求：无资源请求

**框架输出**：

- 汇报类型：同步型
- 结构选择：结论先行+简略事实
- 决策点：了解本季度经营结果（无拍板项）
- 风险预判：无
- 资源需求：无

**银行场景注意**：季度汇报中，领导可能追问数字来源，提前准备好数据来源页。

## 示例二：问题升级要资源（请求型+结论先行）

**场景**：某客户项目遇到技术障碍，需要领导协调IT部门支持，预计需要2周时间和1个开发人员。

**判断过程**：

1. 判断类型：不汇报项目会延期 → 请求型（需要资源）
2. 决策点数量：1个（是否批准IT资源投入）
3. 质疑预判：
   - 资源质疑（为什么需要这么多？）
   - 优先级质疑（这事比别的项目重要吗？）
4. 资源需求：明确（2周+1人），可量化

**框架输出**：

- 汇报类型：请求型
- 结构选择：结论先行
- 决策点：是否批准IT资源投入
- 风险预判：
  - 质疑1：资源量 → 准备项目延期影响说明
  - 质疑2：优先级 → 准备项目价值和紧迫性说明
- 资源需求：明确量化（2周，1人）

## 示例三：方案审批汇报（说服型+SCQA）

**场景**：客户提出一个非标方案，超出常规授权，需要领导特批。领导原来的态度是倾向保守处理。

**判断过程**：

1. 判断类型：需要改变领导原有判断 → 说服型
2. 决策点数量：1个（是否批准非标方案）
3. 质疑预判：
   - 风险质疑（这事有什么风险？）
   - 逻辑质疑（为什么非标是合理的？）
4. 结构选择：SCQA（需要铺垫逻辑改变领导判断）

**框架输出**：

- 汇报类型：说服型
- 结构选择：SCQA
- 决策点：是否批准非标方案
- 风险预判：
  - 风险质疑 → 准备风险对应对和止损机制
  - 逻辑质疑 → 准备为什么标准方案不适用的说明
- 资源需求：无额外资源请求

## 示例四：风险预警汇报（结论先行+风险对应对）

**场景**：贷后监控发现某客户现金流异常，需要领导关注，但不需要当场做实质决策。

**判断过程**：

1. 判断类型：预警性同步，可能后续需要拍板 → 同步型（带决策点）
2. 决策点数量：2个
   - 决策1：是否提高监控频率
   - 决策2：是否需要提前介入
3. 质疑预判：
   - 数据质疑（这数据准确吗？）
   - 行动质疑（现在就要动吗？）
4. 结构选择：结论先行+风险对应对

**框架输出**：

- 汇报类型：同步型（带决策点）
- 结构选择：结论先行+风险对应对
- 决策点：
  - 主：是否提高监控频率
  - 次：是否需要提前介入（可延后决策）
- 风险预判：
  - 数据质疑 → 准备数据来源说明
  - 行动质疑 → 准备"如果不动会怎样"的说明
- 资源需求：暂无

## 示例五：汇报前判断失当的调整

**场景**：准备了一个结论先行的请求型汇报，但领导听完第一句话就问"背景是什么"。

**调整过程**：

1. 信号识别：领导在结论后追问背景
2. 判断：这不是时间紧的领导，或者这个议题领导不熟悉背景
3. 调整：下次遇到这种情况，改为SCQA或加一页背景铺垫

**框架输出**：

- 原判断：请求型+结论先行
- 触发信号：领导追问"背景是什么"
- 调整后策略：加一页背景铺垫，或改用SCQA
- 适用场景：领导不熟悉的议题、新业务场景、跨部门议题
