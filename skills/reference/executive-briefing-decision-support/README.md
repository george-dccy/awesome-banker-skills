# 向上汇报与决策支持判断框架

## 这是什么

在向领导汇报之前，你需要先判断：
- 这件事领导需要"知道"还是"拍板"
- 用哪种汇报结构
- 哪些是真正的决策点
- 领导可能会质疑什么

本 skill 提供这套判断框架，而不是具体的汇报稿。

## 公开方法论来源

本 skill 的核心框架改编自以下公开管理咨询方法：

1. **Minto Pyramid Principle**（麦肯锡芭芭拉·明托《金字塔原理》）
   - 核心：先说结论，再说支撑
   - 用途：请求型汇报的结构选择

2. **BCG 决策框架**
   - 核心：识别"决定—风险—资源"三角
   - 用途：决策点识别和资源需求评估

3. **电梯演讲（Elevator Pitch）**
   - 核心：30 秒内说清一件事
   - 用途：同步型汇报的结构基准

4. **SCQA 框架**（Situation-Complication-Question-Answer）
   - 背景→冲突→问题→答案
   - 用途：背景先行汇报的结构选择

## 与其他资产的关系

```
executive-briefing-decision-support（判断框架）
    │
    ├──→ 决定用哪种结构 → skill.reference.decision-brief-framework（五段式模板）
    │
    └──→ 需要写具体汇报 → skill.action.report-decision-brief（执行脚本）
```

## 适用场景

- 季度工作汇报
- 项目进展同步
- 问题升级请求资源
- 方案提请拍板
- 风险预警汇报

## 非适用场景

- 日常闲聊式同步
- 只需要领导"知道"但无决策需求的事项
- 还没有完成基础事实采集的场景
