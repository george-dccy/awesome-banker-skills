# Knowledge Routing

## 光大产品路由

### 结算底座

- **命中条件**：客户卡点在账户、收付、回单、对账、付款流程、授权管理
- **知识包**：`pack.banks.ceb.corporate-settlement.basic-settlement`
- **适用场景**：基础结算需求；卡点在收付款执行层面而非融资层面
- **边界提示**：结算产品通常边界较宽，重点看企业是否在可服务范围内

### e付通（供应链协同）

- **命中条件**：出现核心企业、供应商、应收账款、1+N 保理、融e链、线上化确认、供应链融资、账期占压、交易链
- **知识包**：`pack.banks.ceb.transaction-banking.yangguang-e-pay`
- **适用场景**：客户处于核心企业-供应商交易链中，卡点在应收账款确认效率、融资通道或线上化协同
- **行业触发**：制造业供应链、建筑工程分包、零售分销、大宗商品贸易
- **边界提示**：需确认核心企业准入情况；供应链融资涉及贸易背景真实性核查

### 电费通（电费证）

- **命中条件**：出现电费、电费缴纳、电网、电费结算单、国内证、福费廷、电费支付优化、月度缴费
- **知识包**：`pack.banks.ceb.trade-finance.yangguang-electricity-certificate`
- **适用场景**：客户有持续、大额电费支出，卡点在缴费节奏与资金占用优化
- **行业触发**：高耗能制造（化工、钢铁、有色金属）、数据中心、大型园区、商业综合体
- **边界提示**：电费证业务涉及国内证开证条件；关注客户与电网的结算周期

### 阳光供应链

- **命中条件**：保函、保理、供应链平台、国际结算、信用证
- **知识包**：`pack.banks.ceb.supply-chain.yangguang-supply-chain`
- **适用场景**：需要担保、应收账款买断、跨境结算、供应链平台接入
- **边界提示**：保函涉及银行或有负债；保理涉及应收账款确权

### 普惠场景金融

- **命中条件**：小微企业、个体工商户、涉农、灵工、物流、安居、数字政务场景
- **知识包**：`pack.banks.ceb.inclusive-finance.scenario-map`
- **适用场景**：普惠口径客户需要快速融资、场景嵌入、线上化审批
- **边界提示**：普惠贷款有额度和资质要求；场景金融涉及特定场景合规性

## 通用能力路由

- 客户需求诊断结论需进一步验证 -> `pack.common.sales.consultative-b2b`
- 客户处于犹豫或异议阶段 -> `pack.common.psychology.business-communication`

## 路由优先级

1. **场景特异性优先**：产品知识包（结算 / e付通 / 电费通 / 供应链）优先于通用能力包
2. **产品知识包内**：按场景特异性选主包，不明确时按回退逻辑
3. **多包命中**：最多选 1 个产品主包 + 1 个产品备选包 + 1 个通用能力包

## 回退逻辑

- 命中供应链/核心企业/应收账款场景但未明确产品 -> 默认走 e付通
- 命中电费/能源缴费场景但未明确产品 -> 默认走电费通
- 命中基础收付/账户问题 -> 走基础结算
- 无法判断场景类型 -> 走基础结算，并在输出中标注"需进一步厘清客户站位"
- 命中小微/普惠但场景不明确 -> 走普惠场景地图

## 读取建议

1. 先读取产品主包 `README.md` 与 `modules/*`
2. 再读取产品主包 `faq.md`，用于边界快速确认
3. 如有备选产品包，补读其 `README.md` 和 `faq.md`
4. 最后补通用能力包，提升适配判断质量

## 多包场景处理

- 命中多个产品包时，按场景特异性选主包，另一个作为备选
- 输出时明确"主推方向"与"备选方向"
- 如需跨条线协同，明确标注并说明协同边界
