# 事项状态评估模型

## 五级状态定义

### 1. 待对方回应（Waiting）

**定义**：我方已输出，等待对方反馈。

判断信号：

- 已发送方案，等待客户确认
- 已提出问题，等待客户回复
- 已约时间，等待客户确认

处理原则：

- 明确等待内容，设置最晚提醒时间
- 不重复催促，除非超过约定时间窗口
- 记录等待开始时间，用于后续节奏判断

### 2. 进行中（In Progress）

**定义**：双方都在推进，有明确进展。

判断信号：

- 对方正在内部审批
- 我方正在准备材料
- 已进入办理流程但尚未完成
- 阶段性成果已交付，等待下一阶段

处理原则：

- 保持节奏，不松懈
- 记录最新进展时间
- 识别下一障碍点

### 3. 遇阻（Blocked）

**定义**：存在明确障碍，无法按预期推进。

判断信号：

- 对方反馈需要重新评估
- 内部条件不满足
- 外部政策或市场条件变化
- 多次催促无实质回应

处理原则：

- 识别阻力的根本原因
- 判断是否可以绕过或替代
- 决定是继续推还是调整预期

### 4. 已闭环（Closed）

**定义**：事项完成，结果已同步对方。

判断信号：

- 对方书面或口头确认接收
- 相关流程已完结
- 结果已记录并存档

处理原则：

- 记录闭环时间
- 确认是否需要后续跟进（如执行阶段）
- 沉淀经验到 case-notes

### 5. 已失效（Invalid）

**定义**：事项不再适用或对方已选择其他路径。

判断信号：

- 客户明确表示不需要
- 对方选择了竞争对手
- 市场或政策环境使事项不再成立
- 时间窗口已关闭

处理原则：

- 记录失效原因
- 不纠缠，礼貌收尾
- 如有替代方案，评估是否值得提出

## 评估流程

1. **逐项过堂**：对每个事项按上述五级定义判断当前状态
2. **找异常**：识别超过预期时间仍未推进的事项
3. **识别风险**：标注遇阻和长期待对方回应的事项
4. **形成状态表**：输出每个事项的状态和关键时间点
