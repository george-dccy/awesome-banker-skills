# 动作推动模型

## 核心逻辑

推动事项进展的核心是：**明确障碍点 → 选择正确动作 → 匹配推动对象**

不是所有事项都适合用同一种方式推动。根据事项状态和障碍类型，选择对应的推动动作。

## 三类障碍 × 对应动作

### 障碍类型一：对方在等（Passive Waiting）

**特征**：对方没有明确拒绝，但没有主动推进。

推动策略：

- **升温策略**：增加触达频率或触达层次
  - 补充新信息或新视角重新引起关注
  - 升级触达对象（如从对接人升级到客户高层）
- **简化策略**：降低对方配合成本
  - 把复杂流程拆成简单步骤
  - 提供更明确的材料和模板
- **共识策略**：重新明确双方的共同目标
  - 提醒事项对对方的价值
  - 重新对齐预期和时间点

### 障碍类型二：对方有顾虑（Active Hesitation）

**特征**：对方有明确担忧或反对意见。

推动策略：

- **倾听策略**：先理解顾虑本质
  - 询问具体担忧是什么
  - 不急于反驳，先共情
- **回应策略**：针对性解答顾虑
  - 提供证据或案例（公开可引用的）
  - 调整方案以消除顾虑
  - 提供保障或备选方案
- **时间策略**：设置明确决策时间点
  - 说明时间窗口和后果
  - 给对方一个明确的回复期限

### 障碍类型三：外部条件不满足（External Block）

**特征**：审批、政策、市场条件、资源等外部因素导致无法推进。

推动策略：

- **替代策略**：寻找替代路径
  - 是否有其他产品或方案可以达成类似目标
  - 是否可以调整条件以满足要求
- **等待策略**：明确等待什么条件
  - 设置条件触发点
  - 在等待期间保持关系温度
- **上推策略**：评估是否需要升级处理
  - 判断是否有内部资源可以协助解决
  - 是否需要领导或跨部门协调

## 推动动作类型清单

### 触达动作

- 发送跟进邮件/微信
- 电话沟通
- 拜访
- 发起线上会议
- 提交正式材料

### 信息动作

- 补充方案细节
- 提供案例参考
- 更新进展状态
- 提供市场或政策信息

### 协调动作

- 内部资源协调
- 跨部门沟通
- 领导支持请求
- 第三方配合推动

### 决策动作

- 请求客户确认
- 请求客户做选择
- 提供方案A/B
- 设置决策期限

## 动作选择决策树

```
事项当前状态是？
│
├─ 待对方回应
│   ├─ 超过约定时间 → 升温触达 / 升级触达对象
│   └─ 未超过约定时间 → 保持节奏，等待时机
│
├─ 进行中
│   ├─ 有明确障碍 → 识别障碍类型，选择对应策略
│   └─ 无障碍，进展正常 → 保持节奏
│
├─ 遇阻
│   ├─ 对方有顾虑 → 倾听 + 回应 + 时间策略
│   ├─ 外部条件不满足 → 替代 / 等待 / 上推
│   └─ 多次无实质回应 → 升级触达 / 调整预期
│
└─ 长期待对方回应（超过预期）
    ├─ 重新评估事项价值和紧迫度
    └─ 决定继续推还是标记为低优先级
```

## 推动节奏参考

| 事项状态 | 建议推动节奏 |
|---------|------------|
| 待对方回应（未超期） | 按约定时间等待，可发温馨提示 |
| 待对方回应（已超期） | 1-2天内触达，升温推动 |
| 进行中，正常 | 保持正常节奏 |
| 进行中，有小障碍 | 3-5天内解决 |
| 遇阻 | 先评估障碍类型，再决定推动节奏 |
| 长期待对方回应 | 重新评估，必要时降级或搁置 |
