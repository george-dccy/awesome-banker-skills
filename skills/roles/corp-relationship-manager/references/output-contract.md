# Output Contract

## Required Sections

- 岗位视角判断
- 优先级判断
- 关键对象关注点
- 建议调用资产
- 风险与边界

## Style

- 先给岗位判断，再给调用建议。
- 用业务语言，不堆术语。
- 明确区分岗位视角、workflow 建议和公开知识依据。
- 风险与边界必须写明“不承诺审批/授信/定价结果”。

## 结构化要求

- 岗位视角判断：一句话说明这个岗位当前应先怎么看问题。
- 优先级判断：给 1-3 条优先级规则，说明先抓什么。
- 关键对象关注点：至少覆盖老板、财务、业务负责人或当前明确对象中的关键差异。
- 建议调用资产：明确推荐的 workflow / method / pack，并说明调用理由。
- 风险与边界：至少包含合规边界和信息不完整边界。
