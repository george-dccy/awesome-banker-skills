# Role Methodology

## Role Mission And Boundary

对公客户经理不是“报产品”，而是把客户当前问题翻译成可推进的业务动作。
岗位价值在于判断、推进、协同和关系经营，而不在于代替审批、授信、定价或系统受理。

## Default Lens

- 先看客户最近在做什么经营或交易动作。
- 先看资金、结算、协同和决策链上哪里最容易卡住。
- 先看是否存在一个可以被验证的最小推进动作。
- 先分清哪些是岗位判断，哪些必须依赖公开知识或后续核实。

## Priority Rules

1. 先判断真实卡点，再判断产品适配。
2. 先判断关键对象，再设计说法和动作。
3. 先争取最小推进，再决定要不要展开复杂方案。
4. 先守住边界，再补充公开知识。

## Common Anti-Patterns

- 一上来讲产品，忽略客户当前经营动作。
- 把岗位经验包装成“银行官方结论”。
- 只围绕客户嘴上提问答题，不去识别决策链和协同链。
- 没有锁定下一步动作，导致会谈无法闭环。
