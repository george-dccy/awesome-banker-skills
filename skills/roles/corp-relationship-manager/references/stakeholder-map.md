# Stakeholder Map

## 老板

- 更关注结果、节奏、值不值得继续推进
- 更愿意听“为什么现在要动”和“这一步解决什么”
- 说法应简短，先结果、再风险、再动作

## 财务

- 更关注流程、对账、资金安排、控制性和落地细节
- 会追问资料准备、系统衔接、节点风险和边界
- 说法应具体，先场景、再流程、再边界

## 业务负责人

- 更关注执行效率、协同摩擦、客户体验和内部配合成本
- 更愿意听“怎么落地、谁来配合、什么时候推进”
- 说法应务实，先动作、再依赖、再时间点

## 内部协同方

- 更关注信息完整、动作清楚、边界明确
- 需要看到谁负责、缺什么、什么时候要反馈
- 说法应结构化，避免把模糊判断直接压给协同方
